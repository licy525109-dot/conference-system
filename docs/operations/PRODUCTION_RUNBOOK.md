# Production Runbook

## 1. 每天运营检查

建议每天固定检查：

1. 打开后台 `数据看板`，查看今日收入、今日订单、待支付订单、今日报名。
2. 打开 `订单支付`，筛选 `只看异常`，处理新增异常订单。
3. 打开 `报名名单`，确认新增报名记录、核销进度和后台备注。
4. 打开 `操作日志`，查看导出、核销、备注修改和异常处理是否符合预期。
5. 如有线上活动当天，额外检查 API 健康：`curl http://127.0.0.1:3001/api/health`。

## 2. 如何查看报名

后台路径：`报名名单`。

常用筛选：

- 会议：选择目标会议。
- 关键词：报名号、姓名、手机号、订单号。
- 报名状态：已确认、已取消、已退款。
- 核销状态：待核销、已核销、无需核销。

查看详情：

- 点击 `详情`。
- 查看参会人、手机号、单位、核销状态、报名表单。
- 如需记录客服沟通结论，使用 `内部备注`。

## 3. 如何导出报名

后台路径：`报名名单`。

步骤：

1. 设置当前筛选条件。
2. 点击右上角 `导出 CSV`。
3. 用 Excel 或表格软件打开 CSV。
4. 核对导出行数和筛选条件。

注意：

- 金额列单位是元。
- 导出动作会记录到 `操作日志`。
- 不会导出密钥、token、AppSecret、商户私钥等敏感配置。

## 4. 如何导出订单

后台路径：`订单支付`。

步骤：

1. 设置会议、订单状态、支付状态或关键词。
2. 如只处理异常，勾选 `只看异常`。
3. 点击右上角 `导出 CSV`。

注意：

- 金额列单位是元。
- 后台订单页不提供手动修改支付状态。
- 导出动作会记录到 `操作日志`。

## 5. 如何处理支付异常

后台路径：`订单支付`。

步骤：

1. 勾选 `只看异常`。
2. 点击订单 `详情`。
3. 阅读 `异常识别` 中的原因。
4. 对照微信商户后台、用户反馈、报名名单进行人工核对。
5. 在异常处理备注中记录结论。
6. 点击 `记录处理备注`。

当前允许的处理：

- 记录人工核对结论。
- 联系用户重新支付或重新报名。
- 联系开发排查数据不一致。

当前不允许的处理：

- 不在后台直接改订单为已支付。
- 不在后台直接触发自动退款。
- 不在后台重新执行支付成功幂等逻辑。

## 6. 如何核销报名

后台路径：`报名名单`。

步骤：

1. 搜索报名号、姓名或手机号。
2. 点击 `详情`。
3. 在参会人表格中确认身份信息。
4. 点击 `确认核销`。
5. 二次确认后完成核销。

注意：

- 只有已支付确认且待核销的参会人可核销。
- 重复核销会被拒绝。
- 核销动作会记录到 `操作日志`。

## 7. 如何查看操作日志

后台路径：`系统管理 / 操作日志`。

可筛选：

- 关键词：操作人、对象、摘要。
- 动作：登录、创建、更新、删除、系统。
- 对象类型：会议、票种、报名、参会人、订单导出、报名导出、支付异常、后台账号。

常见对象类型：

- `AdminUser`：后台登录或账号操作。
- `Conference`：会议创建、编辑、上下架。
- `RegistrationSku`：票种操作。
- `FormField`：报名字段操作。
- `Registration`：报名备注。
- `RegistrationAttendee`：核销。
- `Order`：订单导出。
- `PaymentException`：异常订单人工处理备注。

## 8. 用户反馈“已支付但没报名”排查

按以下顺序排查：

1. 在 `订单支付` 搜索用户手机号、订单号或商户单号。
2. 查看订单是否已支付。
3. 查看订单详情是否出现 `订单已支付，但报名记录缺失`。
4. 在微信商户后台核对交易号、金额、支付时间。
5. 在 `报名名单` 搜索手机号和订单号，确认是否存在报名记录。
6. 在订单详情中记录人工处理备注。
7. 如果微信已支付且后台缺报名，不要直接改数据库，联系开发处理。

联系开发时提供：

- 订单号。
- 商户单号。
- 微信交易号。
- 用户手机号。
- 实付金额。
- 支付时间。
- 后台异常原因截图或文字。

## 9. 导出失败处理

常见原因：

- 登录已过期。
- 筛选结果过大或网络中断。
- API 服务异常。

处理步骤：

1. 刷新后台并重新登录。
2. 缩小筛选范围，例如按会议或状态导出。
3. 检查 API 健康：`curl http://127.0.0.1:3001/api/health`。
4. 查看 PM2 日志：`pm2 logs conference-api --lines 100`。
5. 若仍失败，联系开发并提供筛选条件、时间点和错误提示。

## 10. API 异常检查

服务器检查命令：

```bash
curl http://127.0.0.1:3001/api/health
pm2 status conference-api
pm2 logs conference-api --lines 100
```

注意：

- API 真实端口是 3001。
- 3000 是 certificate-site，不是会议 API。
- 不要打印 `.env.production` 内容。

## 11. 什么时候联系开发

出现以下情况应联系开发：

- 微信商户后台显示已支付，但后台订单不是已支付。
- 后台订单已支付，但报名记录缺失。
- 支付金额和订单金额不一致。
- 重复回调、重复报名、重复核销等幂等问题。
- 导出连续失败且 API health 不正常。
- 需要退款、发票或批量数据修复。
- 需要执行数据库迁移或生产数据脚本。

联系开发前准备：

- 发生时间。
- 后台页面和筛选条件。
- 订单号、报名号、手机号。
- 微信交易号或商户单号。
- 操作日志中的相关记录。
