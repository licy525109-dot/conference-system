# 会议报名缴费 MVP 测试计划

## 1. 测试目标

验证会议报名缴费 MVP 的核心闭环：

- 会议可浏览。
- 规格价格正确。
- 动态表单可配置并校验。
- 订单由后端重新计价。
- mock 支付可完成闭环。
- 微信支付预下单与回调满足安全要求。
- 支付成功后生成报名记录。
- 重复回调不重复生成报名记录。
- 后台接口必须鉴权。

## 2. 测试范围

### 覆盖

- 后端 API。
- Prisma 数据模型和约束。
- 支付成功事务。
- 支付回调幂等。
- 用户端 H5。
- 微信小程序基础页面。
- 后台管理端。
- 安全和密钥泄露检查。

### 不覆盖

以下功能不在 MVP 测试范围，因为不实现：

- 优惠券
- 会员
- 商城
- AI 知识库
- 房间座位酒店
- 抖音 / 小红书 / B 站原生小程序
- 退款
- 发票
- 签到核销

## 3. 后端单元测试

### 3.1 计价和订单

- SKU A 价格为 100000 分，quote 返回 100000。
- SKU B 价格为 70000 分，quote 返回 70000。
- 创建订单时前端传入伪造金额，后端忽略。
- 创建订单时从数据库读取 SKU.priceCent。
- Order.originAmountCent 等于服务端 SKU 价格。
- Order.payableAmountCent 等于服务端计算结果。
- OrderItem.unitPriceCent 保存创建订单时的 SKU 价格快照。
- 规格停用时不能创建订单。
- 会议未发布时不能创建订单。

### 3.2 动态表单

- 必填字段缺失返回 `VALIDATION_ERROR`。
- phone 字段格式错误返回 `VALIDATION_ERROR`。
- email 字段格式错误返回 `VALIDATION_ERROR`。
- select/radio 值不在 options 内返回 `VALIDATION_ERROR`。
- 未知字段按 API 规范拒绝或忽略，但必须有测试固定行为。
- label 和 options 不允许脚本或 HTML 注入。
- 合法表单提交可以创建订单。

### 3.3 支付成功处理

- pending 订单支付成功后变为 paid。
- 支付成功后创建 Registration。
- Registration.orderId 唯一。
- 已 paid 订单重复处理不会创建重复 Registration。
- 支付金额与订单金额不一致时不更新订单。
- 订单不存在时不创建 Payment 和 Registration。

### 3.4 微信支付回调

- 正常成功回调通过。
- 重复成功回调通过且无副作用。
- 金额不一致回调失败。
- 订单不存在回调失败。
- 验签失败回调失败。
- 解密失败回调失败。
- 回调处理不记录密钥和敏感解密原文。

### 3.5 后台鉴权

- 未登录访问 `/api/admin/conferences` 返回 401。
- 未登录访问 `/api/admin/orders` 返回 401。
- 登录成功返回 token。
- 错误密码不能登录。
- 停用管理员不能登录。

## 4. 后端集成测试

### 4.1 报名完整链路

步骤：

1. 查询会议列表。
2. 查询会议详情。
3. 查询报名表单。
4. 调用 quote。
5. 创建订单。
6. mock 支付。
7. 查询我的报名。

期望：

- 订单金额使用分。
- 支付前没有 Registration。
- 支付后有一条 Registration。
- 我的报名能看到报名记录。

### 4.2 重复支付链路

步骤：

1. 创建订单。
2. mock 支付成功。
3. 再次 mock 支付同一个订单。
4. 查询 Registration 数量。

期望：

- 订单保持 paid。
- Registration 仍只有一条。

### 4.3 金额篡改链路

步骤：

1. 前端请求创建订单时附带伪造金额字段。
2. 后端创建订单。
3. 查询订单金额。

期望：

- 订单金额等于数据库 SKU 价格。
- 伪造金额不入库、不生效。

- 微信 AppSecret 未提交。
- 微信商户私钥未提交。
- 微信 API v3 Key 未提交。

## 8. 命令建议

开发完成后按阶段运行：

```bash
pnpm typecheck
pnpm lint
pnpm test
pnpm prisma:migrate
```

本地服务验收：

```bash
docker compose up -d postgres redis
pnpm dev:api
pnpm dev:user:h5
pnpm dev:admin
```

```bash
pnpm dev:user:mp-weixin
```

然后使用微信开发者工具打开生成目录进行验证。
