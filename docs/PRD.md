# 会议报名缴费 MVP PRD

## 1. 背景

本项目第一阶段目标是快速上线一个会议报名缴费闭环。用户可以查看会议信息、选择报名规格、填写报名信息、创建报名订单并完成支付；支付成功后系统生成正式报名记录。后台管理员可以维护会议、报名规格、动态报名表单，并查看订单和报名名单。

MVP 必须小而可交付，所有金额和支付相关逻辑以后端为准。

## 2. 项目目标

- 用户能在 H5 和微信小程序查看会议首页。
- 用户能进入会议详情页，看到会议介绍、时间地点、报名规格和价格。
- 用户能选择报名规格并填写后台配置的动态报名表单。
- 用户能创建报名订单。
- 开发环境支持 mock 支付。
- 预留并实现微信支付 API v3 的预下单和回调处理能力。
- 支付成功后生成报名记录。
- 用户能查看我的报名。
- 后台管理员能完成最小管理操作。

## 3. MVP 功能范围

### 3.1 用户端

- 会议首页
  - 展示已发布会议列表。
  - 展示会议标题、封面、时间、地点、报名状态。
- 会议详情
  - 展示会议基础信息、详情内容、报名时间、会议时间、地点。
  - 展示可报名规格。
- 报名规格
  - 每个会议可配置多个报名规格。
  - 规格包含名称、说明、价格、排序、启用状态。
  - 所有金额使用整数分。
- 动态报名表单
  - 表单字段由后台按会议配置。
  - MVP 支持字段类型：text、phone、email、select、radio、date、textarea。
  - 前端校验只用于体验，后端校验为准。
- 报名订单
  - 用户选择会议和规格后创建订单。
  - 后端创建订单时必须重新读取规格价格并计价。
  - quote 接口只用于展示，不作为实际支付金额依据。
- mock 支付
  - 开发环境可通过 mock 支付完成订单。
  - mock 支付成功后触发与真实支付成功一致的订单和报名记录流程。
- 微信支付预下单与回调
  - 支持微信支付 API v3 预下单接口封装。
  - 支持支付回调验签、解密、金额校验、事务处理、幂等处理。
- 我的报名
  - 用户能查看已支付成功并生成的报名记录。
  - 展示会议、规格、报名人、手机号、支付金额、报名状态。

### 3.2 后台管理

- 管理员登录。
- 会议管理：创建、编辑、发布、下架会议。
- 报名规格管理：创建、编辑、启用、停用规格。
- 表单字段管理：配置字段类型、字段名、是否必填、选项、排序。
- 订单管理：查看订单列表、订单详情、支付状态。
- 报名名单：查看报名记录，按会议筛选。
- 后台接口必须鉴权。

## 4. 明确暂不实现

以下能力不进入 MVP，不做 UI、不做业务流程、不做数据库完整实现：

- 优惠券
- 会员
- 商城
- AI 知识库
- 房间座位酒店
- 抖音 / 小红书 / B 站原生小程序

以下能力也不进入第一阶段：

- 自动退款
- 发票
- 签到核销
- 复杂页面装修
- 复杂库存锁定策略
- 多级分销或渠道归因

如未来需要扩展，只允许在 MVP 中保留低成本扩展点，例如订单备注字段、支付 provider 字段、会议内容 JSON 字段，不实现完整业务。

## 5. 用户角色

### 5.1 普通用户

- 浏览会议。
- 查看会议详情。
- 选择报名规格。
- 填写报名信息。
- 创建订单并支付。
- 查看我的报名。

### 5.2 后台管理员

- 登录后台。
- 管理会议内容。
- 管理报名规格。
- 管理动态报名表单。
- 查看订单。
- 查看报名名单。

## 6. 核心业务流程

### 6.1 报名支付流程

1. 用户进入会议详情。
2. 前端获取会议详情、报名规格、动态表单字段。
3. 用户选择报名规格并填写报名信息。
4. 前端调用 quote 接口展示预计金额。
5. 用户确认报名。
6. 前端提交会议 ID、规格 ID、报名表单数据。
7. 后端重新读取会议、规格和字段配置。
8. 后端校验会议状态、报名时间、规格状态、表单数据。
9. 后端重新计算订单金额，创建 pending 订单。
10. 用户发起 mock 支付或微信支付。
11. 支付成功后，后端更新订单为 paid。
12. 后端在同一事务中生成 Registration。
13. 用户查看支付结果和我的报名。

### 6.2 支付回调流程

1. 微信支付平台请求 notify 接口。
2. 后端验签。
3. 后端解密回调报文。
4. 后端根据 outTradeNo 查询本地订单。
5. 后端校验支付金额必须等于订单 payableAmountCent。
6. 后端开启事务。
7. 如果订单已 paid，直接返回成功，不重复创建报名。
8. 如果订单为 pending，则更新 Payment、Order，并创建 Registration。
9. 返回微信支付成功响应。

## 7. 业务规则

- 所有金额字段使用整数分，禁止浮点数金额。
- 前端提交的金额不可信。
- quote 只用于展示，创建订单时必须后端重新计价。
- 支付成功前不能生成 Registration。
- 重复支付回调不能重复生成 Registration。
- 后台接口必须鉴权。
- 真实密钥不得写入代码、文档、日志、测试快照。
- 支付日志可以记录 orderNo、outTradeNo、status，不记录密钥和敏感解密原文。

## 8. 验收标准

- 会议首页能展示已发布会议。
- 会议详情能展示会议信息、规格和动态表单。
- 不同规格展示不同价格。
- 创建订单时后端重新计价。
- 篡改前端金额不会影响订单金额。
- mock 支付成功后订单变为 paid。
- 支付成功后生成一条报名记录。
- 重复支付回调不会生成重复报名记录。
- 我的报名能看到已支付报名。
- 后台未登录不能访问管理接口。
- 后台能维护会议、规格、表单字段，并查看订单和报名名单。
