本阶段基于当前 `main` 的 Phase 0、Phase 2、Phase 3、Phase 4 报告和现有代码进行上线前代码级质检，覆盖：

## 11. 宝塔部署清单

本阶段修改了用户端、后台前端和文档，未修改后端服务、Prisma、支付和生产配置。

部署建议：

- 不需要执行 Prisma migrate。
- 不需要重启 `conference-api`，也不需要 `pm2 restart conference-api`。
- 不需要修改或创建 `/www/wwwroot/conference-system/.env.production`。
- 需要分别构建用户端 H5 与后台静态资源：
  - `pnpm build:user:h5`
  - `pnpm build:admin`
- 将用户端 H5 构建产物和后台构建产物 rsync 到宝塔对应静态目录。
- 确认生产 API 仍走真实会议 API 端口 `3001`，不要误用 `3000`。
- 若仅合并文档变更而不部署前端静态文件，则无需服务器部署。

静态部署后建议人工检查：

- H5 首页、会议详情、报名页、支付结果页、我的报名。
- 后台登录、Dashboard、会议管理、报名名单、订单列表、页面装修、素材管理。
- 浏览器控制台无明显前端资源加载错误。

## 12. 小程序体验版验证清单

在微信开发者工具中执行：

- 导入 `apps/user/dist/build/mp-weixin`。
- 编译小程序。
- 打开首页并确认会议列表、CMS 首页或空状态正常。
- 进入会议详情并确认票种、库存、CMS 内容、底部报名按钮不遮挡。

- 提交订单，确认创建订单后进入支付结果页。
- 触发微信资料授权弹窗，确认用户可理解授权用途。
- 检查支付入口，mock/微信支付模式按环境配置表现。
- 检查支付结果页成功、待支付、取消或失败状态。
- 打开我的报名，确认已支付生成的报名记录可见。
- 打开自定义 CMS 页面，确认不支持组件不会粗糙展示。
- 打开会员、购物车、商城列表、商品详情，确认灰度/后续开放文案清楚。

## 13. 上线前遗留风险

- 本阶段完成代码级质检和构建验证，但未连接生产真实数据做全链路人工回归。
- 微信支付真实 prepay、notify、金额校验、幂等处理未在本阶段改动，也未做生产支付实测；上线前仍需使用安全测试订单走一遍支付链路。
- 小程序体验版仍需在微信开发者工具和真机上确认授权、支付入口、固定底部按钮和页面滚动不遮挡。
- 后台生产账号权限需要人工确认，避免运营账号缺少会议、报名、订单、页面装修等主线权限。
- 构建产物 chunk size 警告仍存在，不阻断上线，但后续可考虑按路由拆包。
