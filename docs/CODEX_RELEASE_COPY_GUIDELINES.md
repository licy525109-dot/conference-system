# Codex Release Copy Guidelines

## 1. 报名状态

统一使用面向用户可理解的状态词，不直接暴露内部枚举：

| 场景 | 推荐文案 | 说明 |
| --- | --- | --- |
| 可报名 | 报名中 | 可作为主状态，CTA 可显示“立即报名” |
| 名额较少 | 名额紧张 | 只用于库存确实偏低的展示，不替代后端库存校验 |
| 截止报名 | 已截止 | 不暗示还可以提交订单 |
| 会议结束 | 已结束 | 会议时间已结束时使用 |
| 报名成功 | 报名成功 | 支付成功并生成报名记录后使用 |

## 2. 支付状态

| 场景 | 推荐文案 | 操作建议 |
| --- | --- | --- |
| 支付成功 | 支付成功 | 主操作为“查看我的报名” |
| 支付处理中 | 支付处理中 | 提供“刷新状态” |
| 待支付 | 待支付 | 可提供“继续支付” |
| 支付失败 | 支付失败 | 可提供“重新支付” |
| 已取消 | 已取消 | 可提供“返回首页”或“重新报名” |

## 3. 订单状态

| 场景 | 推荐文案 |
| --- | --- |
| 已创建未支付 | 待支付 |
| 支付完成 | 已支付 |
| 支付异常 | 支付异常，请人工核对 |
| 已关闭或取消 | 已关闭 |

订单金额展示必须使用整数分转元的展示工具，页面不得把前端 quote 当作可信最终金额。

## 4. 空状态

空状态遵循“当前结果 + 下一步动作”的结构：

- 无会议：暂无可报名会议，可以稍后再来查看。
- 无报名：还没有报名记录，去看看当前开放的会议。
- 无订单：暂无订单记录。
- 无素材：暂无素材，可以先上传会议封面、嘉宾头像或装修图片。

不要用“没有数据”“空列表”等开发口吻。

## 5. 错误状态

错误提示遵循“发生了什么 + 用户可以做什么”的结构：

- 加载失败：内容加载失败，请稍后重试。
- 提交失败：提交失败，请检查信息后重试。
- 支付异常：支付状态暂未确认，请刷新状态或联系工作人员。
- 无权限：当前账号暂无此操作权限，请联系管理员开通。

不要暴露接口路径、堆栈、数据库错误或支付密钥相关信息。

## 6. 扩展能力状态

| 状态 | 推荐标签 | 推荐说明 |
| --- | --- | --- |
| 已可用 | 已可用 | 可正常使用当前功能 |
| 灰度中 | 灰度中 | 能力正在试运行，建议运营谨慎配置 |
| 后续开放 | 后续开放 | 当前不提供完整链路，后续版本完善 |
| 预留能力 | 预留能力 | 已有入口或展示，不代表正式上线 |
| 对账辅助 | 对账辅助 | 只用于运营核对，不等同完整财务系统 |

会员、商城、财务、优惠券、满减等不属于第一版会议报名缴费主链路，应持续标注扩展状态。

## 7. 灰度/后续开放说明

推荐统一句式：

- 会员：会员权益展示中，暂不参与报名定价。
- 商城：商城试运行中，商品支付和履约后续开放。
- 购物车商品：商品支付后续开放，暂不支持商品结算。
- 财务：对账辅助，不等同完整财务系统。
- 营销配置：营销配置为灰度能力，不改变 quote、下单和支付金额计算逻辑。

避免使用“立即购买”“马上支付”“自动享受会员价”等会让用户误解完整链路已经上线的文案。

## 8. 金额以后端为准说明

用户端报名页、详情页、费用摘要和营销配置相关页面统一使用：

> 金额以提交订单时系统计算结果为准。

涉及 quote 时可补充：

> 前端 quote 仅用于展示。

后台营销配置和会议配置可使用：

> 订单金额仍由后端重新计算。

不得出现“前端计算最终金额”“以页面显示金额为准”等表达。

## 9. 客服/联系提示

推荐句式：

- 如报名信息需要调整，请联系会议工作人员处理。
- 如支付状态长时间未更新，请保留订单号并联系工作人员核对。
- 如页面配置无法发布，请检查不支持组件、图片地址和跳转路径。

客服提示不要承诺自动退款、自动改价、自动改报名状态。

## 10. 危险操作二次确认

危险操作包括下架会议、删除配置、手动核销、调整账号权限等。

推荐结构：

- 标题：确认{操作名称}
- 正文：说明影响范围和不可逆风险。
- 取消按钮：取消
- 确认按钮：确认{操作名称}

示例：

- 确认手动核销：手动核销后将影响参会入场状态，请确认已核对报名人信息。
- 确认下架会议：下架后用户端将不再展示该会议，已产生的订单和报名记录不会删除。
- 确认调整权限：权限变更会影响该账号可访问的后台页面，请确认后继续。
